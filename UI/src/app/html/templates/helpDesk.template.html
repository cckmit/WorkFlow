How to request Toolchain Access

My identity Portal URL : myidentity.travelport.com

LDAP Roles: Please select required roles from the drop down list. This will be subject to your manager approval.

Steps:

1. Select "MyIdentity Access"
2. Select "Request Access"
3. Click Requested Service -> Linux server access option
4. Click Access Type -> Permanent Access option
5. Select "Development" from the Environment drop-down menu, then it'll display all the existing access group names associated with this LDAP id. Need to have your LDAP id created first (for developer roles, use id that is used to login to GVM/VM3/VM4/DL2 eg: e038039)
6. Select "Additional Access for Existing Login Request" from the "Linux Request Type" choice
7. Select "Full Access" from the "Permission" choice
8. Select login shell as "/bin/bash" in selection(default)
9. Enter "zTPF" in search filer and then select required roles from the "Role(s)/Group(s)" dropdown menu.
LDAP Group Name Remarks
tpfsys (RA_TP_USERS) Access to Impact Analysis tool and for all Developer activities in Toolchain(Travelport: MUST HAVE for Impact Analysis & zTPFGI)
ztpfdevtools_dev_appsupport (WF_TP_DEVELOPER) Developer Role (Travelport)
ztpfdevtools_dev_developer_lead (WF_TP_LEAD) Lead Role (Travelport)
ztpfdevtools_dev_managers (WF_TP_DEV_MANAGER) Dev Manager Role(Travelport)
ztpfdevtools_dev_peer_review (WF_TP_PEER_REVIEWER) Peer Reviewer Role(Travelport)
ztpfdevtools_dev_qualitymgmt (WF_TP_QA) QA Role(Travelport)
ztpfdevtools_prod_serveraccess (WF_TP/DL_PROD_SERVER_ACCESS) Toolchain Server Access (Travelport & Delta: MUST HAVE for all Toolchain users)

10. Finally, enter "Access Justification" to submit the form. Once it's approved then he'll get the access.
